# 阶段1：开发环境初始化

**记录时间**: 2026-05

## 1. 环境配置
- **IDE**: 采用 TRAE (v1.0+) 作为核心开发工具，配置 Python 3.x 开发插件。
- **版本控制**: 使用 Git 进行本地版本管理，通过 GitHub Desktop 同步至远程仓库。
- **项目结构**: 采用模块化目录，划定四个核心实验阶段（API、RAG、Web、Agent）。

## 2. 核心工程规范
- **文档先行**: 每个子阶段需包含独立的 README.md，明确说明运行环境及启动命令。
- **依赖透明**: 凡涉及第三方库（如 requests, langchain），必须在对应目录下生成 requirements.txt。

## 3. 注意事项 (Precautions)
- **密钥安全（最高优先级）**: 
  - 严禁将大模型 API Key 以明文形式写入代码。
  - 必须使用 `.env` 环境变量文件管理密钥，并确保该文件已列入 `.gitignore`。
- **环境隔离**:
  - 建议为不同阶段创建独立的虚拟环境（venv），避免包版本冲突。
- **提交规范**:
  - 每次提交代码前需清理临时文件及缓存（如 __pycache__）。
  - Commit Message 需简明描述本次修改的具体功能点。
