本项目旨在通过手写核心逻辑而非依赖高级框架，掌握大模型应用的底层原理。

## Phase 1: 协议通信与环境集成 (API Basics)

**核心目标**：理解 LLM 的无状态本质，掌握 HTTP 协议与参数控制。

- **任务 1.1: 隔离环境与密钥管理** 🔑
  - 在根目录配置 `.env` 隔离敏感信息。
  - 使用 `venv` 构建隔离的 Python 开发环境。
- **任务 1.2: 原始 HTTP 请求实现** 🌐
  - 不使用 SDK，仅用 `requests` 库构造 Header 和 Body，理解 Bearer Token 鉴权。
- **任务 1.3: 响应体解析与 Token 审计** 📊
  - 精准提取 `content` 及消耗的 `completion_tokens`，建立成本意识。
- **任务 1.4: 多轮对话上下文管理** 🧠
  - 手动维护 `messages` 列表，理解 `system`/`user`/`assistant` 角色差异。
- **任务 1.5: 解码概率参数 (Temperature/Top-p)** 🎛️
  - 对比不同参数对输出多样性的影响，理解模型的随机性本质。
- **任务 1.6: 生产级异常处理捕获** 🛡️
  - 针对 401（鉴权失败）、429（超限）、Timeout 等网络异常编写重试与提示逻辑。
- **任务 1.7: 命令行交互产品 (CLI)** 💻
  - 封装一个支持无限循环、可随时退出的中英互译工具。

## Phase 2: 检索增强生成 (RAG Theory & Practice)

**核心目标**：掌握文本向量化与本地知识库挂载，解决模型“幻觉”问题。

- **任务 2.1: 文档预处理与清洗** 📄
  - 实现 PDF/Markdown 文本提取，学习根据语义进行文本切分（Chunking）。
- **任务 2.2: 向量空间映射 (Embedding)** 🔢
  - 调用 Embedding API 将文本转换为高维向量，理解余弦相似度。
- **任务 2.3: 向量数据库实战** 🗄️
  - 使用 ChromaDB 或 FAISS 进行向量存取，实现“语义搜索”而非关键词搜索。
- **任务 2.4: 提示词注入 (Context Injection)** 💉
  - 将检索到的知识片段动态拼入 Prompt，让模型基于事实回答问题。

## Phase 3: 全栈 AI 应用开发 (Full-stack Deployment)

**核心目标**：完成从脚本到产品的跃迁，掌握异步流式通信。

- **任务 3.1: FastAPI 后端封装** ⚡
  - 将 AI 逻辑封装为 RESTful API，掌握 Pydantic 模型验证。
- **任务 3.2: SSE 流式输出实现 (Streaming)** 🌊
  - 实现如同 ChatGPT 一样“逐字蹦出”的效果，理解长连接通信。
- **任务 3.3: 前端交互界面构建** 🎨
  - 使用 Streamlit 或原生 HTML/JS 搭建对话窗口。
- **任务 3.4: 云服务器部署与环境隔离** ☁️
  - 在 Linux 服务器上部署应用，配置反向代理与环境变量保护。

## Phase 4: 智能体与工具调用 (Agent & Tools)

**核心目标**：让 AI 具备“行动力”，从对话工具进化为任务助手。

- **任务 4.1: 函数调用原理 (Function Calling)** 🛠️
  - 定义外部工具格式，让模型自主决定何时调用外部函数（如查询天气）。
- **任务 4.2: 思维链构建 (CoT)** 🔗
  - 引导模型进行分步骤推理，提高解决复杂逻辑问题的准确率。
- **任务 4.3: 简易智能体闭环** 🤖
  - 实现一个具备“自主搜索 -> 汇总信息 -> 生成报告”能力的简易 Agent。
